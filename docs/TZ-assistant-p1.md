# ТЗ: ИИ-помощник СберБизнес (демо) — приоритеты 1–2

Цель: помощник реально **выполняет** команды в демо интернет-банка, а не выдаёт «глухой текст». Все P1 должны работать с обработкой ошибочного/неоднозначного ввода.

Бэкенд: FastAPI, pipeline `services/ai/assistant.py` → `services/banking/queries.py` (+ search, notifications, analytics, tax, services). Фронт: Next.js, `AssistantPanel.tsx`, `StatementView.tsx`, `DocumentDetailView.tsx`, чипы/источники.

---

## P1 — обязательно работают идеально

### 1. Умный поиск (`#1`)
- **Документ по реквизитам:** «Найди счёт от ООО Ромашка за март 2026» → фильтр по контрагенту + дате → карточка `/other/documents/view?doc=`.
- **Платёж по сумме/плательщику:** «Найди платёж на 50 000 от Иванова» → извлечь сумму, найти, показать статус.
- **Контрагент:** «Покажи карточку клиента Петров» → `/services/counterparty?cp=`.
- **Краевые случаи:**
  - Ничего не найдено → подсказки «убрать фильтр по месяцу / изменить сумму / переформулировать».
  - Несколько результатов → нумерованный список + источники для выбора.
  - Год (2026) не считается суммой; стоп-слова (найди/счёт/от/ООО) выкидываются из поиска.
  - Разведение типов: «счёт» → счета/счета на оплату, «платёж» → переводы.

### 2. Единый ассистент-роутер (`#2`)
- Один чат, классификация intent внутри `handle_banking_query` (порядок: выписка → шлюз → источник → налоги → контрагенты → риски → уведомления → быстрые команды → поиск → выписка-уточнение → повтор).
- Максимум один уточняющий вопрос (через `pending_form_fields`).

### 3. Помощь при платеже (`#3`)
- Заполнение формы из чата (rules + LLM tool), подстановка контрагента из БД.
- Валидация УНП (9 цифр), IBAN (BY), лимит org; inline-подсветка полей; блок submit при критичной ошибке.

### 4. Быстрые команды (`#4`) — должны исполняться
- Выписка за период/месяц/квартал/год + по счёту (IBAN/последние цифры/label).
- «Платежи X за март», «Заплати аренду», «Создай платёжку на N для X», «Повтори последний платёж», «Отложенный платёж на DD месяца», «Создай счёт для X», «Покажи последние документы», «Сколько на счёте?», «Расходы за март», «Сравни февраль и март», «Итоги недели/месяца», «Что надо заплатить?», «Реквизиты X», «Добавь контрагента», «Кто не заплатил».

### 5. Умные уведомления (`#5`)
- Баннер + чат, привязка к документу по № в тексте, действие открывает документ/раздел.

### 7–8. Голос (ввод/вывод)
- Web Speech STT (микрофон) → автоотправка; TTS озвучка ответа (Edge fallback без ключей).

### 9. Quick-reply карточки
- Приветственные карточки: остаток, платёж, выписка, найти документ, что заплатить.
- Контекстные чипы после поиска/выписки/баланса/аналитики.

### 13–16, 18. 1С, OCR, источники, сервисы, налоги
- 1С: `/services/onec` (sync, preview, batch import).
- OCR: demo-fallback без ключей, подсветка полей, «Исправь поле X».
- Источники: `?highlight=uuid`/`?doc=uuid` → карточка документа (middleware + хук).
- Сервисы: тарифы, ROI, заявка-модалка.
- Налоги: календарь по типу org, демо-расчёт ФСЗН.

### 21. Стриминг текста
- SSE по фрагментам с сохранением переносов строк (эффект печати).

### 22–24. Персонализация
- Приветствие по названию организации, роль-зависимые карточки, 3D-аватар с эмоцией по тону ответа.
