§02

Почему «на все случаи один голос»

root cause
1
Роутер знает только Inworld
backend/services/tts/voice_router.py на 90 строк. Внутри один if _inworld_configured() — если INWORLD_API_KEY задан, провайдер «inworld», иначе пустой массив. Никакого "google", "qwen", "edge" он не возвращает.
voice_router.py:18-21, :41-71
2
synthesize_speech падает на всём, кроме Inworld
В backend/services/tts/__init__.py:33-39 единственная ветка — if provider == "inworld". Если голос не Inworld (а Google/Qwen/Edge все не Inworld), выбрасывается TtsNotConfiguredError → 503 на /api/tts/speak. Поэтому UI-баннер "API key required" для Google/Qwen висит правильно — бэк их не отдаёт.
__init__.py:33-39
3
Шесть файлов лежат мёртвым грузом
google_tts.py (160 строк, real fallback chain), qwen_voices.py (CosyVoice роутинг), deepgram.py (с auto-fallback), edge_tts.py (бесплатный), gtts_tts.py (мужской через Edge) — у каждого свой synthesize_speech(), который никто не вызывает. Это 600+ строк готового кода, который надо только подключить к роутеру.
backend/services/tts/{google,qwen,deepgram,edge,gtts}_*.py

§05

Что чинить, чтобы открыть 20+ голосов

roadmap
A
Переписать resolve_synthesis_route()
Сейчас 1 ветка, нужен dict {prefix → provider} с проверкой _provider_configured() на каждый и fallback-цепочкой. Голоса ru-RU-* идут в Google, qwen-* в Qwen, aura-2-* в Deepgram, ru-RU-{Dmitry,Svetlana}Neural в Edge, иначе Inworld.
voice_router.py:30-38 → ~50 строк
B
Сделать synthesize_speech() мультипровайдерным
Цепочка ["inworld", "qwen", "google", "deepgram", "edge", "gtts"]. На каждый TtsNotConfiguredError — следующий провайдер. На 401/403/429 (квота) — следующий. На остальные — наверх.
__init__.py:33-39 → ~25 строк
C
list_all_assistant_voices() — собрать все 6 каталогов
Сейчас отдаёт только Inworld. Нужно проверить конфиг каждого провайдера и смерджить его list_assistant_voices() в общий массив groups. Юзер сразу увидит 21 голос в <VoicePicker>.
voice_router.py:41-71 → ~40 строк
D
UI-фишки на бэке готовы, нужны на фронте
tier=neural2/wavenet/standard уже в google_voices.py — фронт может рисовать бейджи Premium / HD / Базовый. Слот 1/2 уже slot-preserving в pickVoiceForGenderPreservingSlot(). Нужно только прокинуть voiceOverride на бэк (users.voice_id), чтобы выбор переживал смену браузера.
store/ttsStore.ts + components/assistant/VoicePicker.tsx
