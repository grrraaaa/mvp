# Workflow демонстрации AI-консультанта (SBBOL Demo)

> **Последовательный сценарий прогона:** вводите команды **по порядку** — каждый шаг опирается на предыдущий и покрывает отдельный блок функционала.  
> Полный каталог (~130 команд) — [ASSISTANT_COMMANDS.md](./ASSISTANT_COMMANDS.md), в UI — `/learning` → вкладка «Команды ИИ».

**Стек:** FastAPI + Next.js 15 + OpenRouter / rule-based fallback.  
**Прод:** https://mvp-beta-umber.vercel.app · **Локально:** http://localhost:3000 (UI) + http://127.0.0.1:8000 (API).

---

## Умеет ли ИИ рисовать диаграммы?

**Да.** Ассистент строит графики по **реальным данным кабинета** (остатки, выписка, аналитика расходов) и показывает их **в чате как PNG** с кнопкой «Скачать». Также есть кнопка 📊 в поле ввода — меню быстрых пресетов.

| Тип | Команда (пример) | Что рисуется |
|-----|------------------|--------------|
| Линейный | `Кассовый прогноз` / `Прогноз остатка` | Остаток на N дней вперёд |
| Круговая | `Расходы за 2026-03` / `Структура расходов` | Доли категорий за месяц |
| Столбчатая | `Сравни февраль и март` | Расходы двух месяцев |
| Гистограмма | `Сколько на счёте?` | Остатки по счетам BYN |
| Каталог | `Покажи график` | Список доступных визуализаций + fallback-график |

**Не умеет:** произвольные схемы, блок-схемы, mindmap «от руки» — только типизированные финансовые диаграммы (`line`, `pie`, `bar`, `donut`).

**Код:** `backend/services/banking/queries.py`, `cash_forecast.py`, `analytics.py` → `charts[]` в ответе → `AssistantChart` + `chartImage.ts` на фронте.

## Перед стартом

```powershell
Invoke-RestMethod http://127.0.0.1:8000/api/health   # ai_mode, tts, db
Invoke-RestMethod http://127.0.0.1:8000/api/tts/status
```

| Символ | Значение |
|--------|----------|
| 🟢 | Обязательный шаг приёмки |
| 🟡 | Расширение (LLM / голос / OCR / TTS) |
| 🔴 | Регрессия — проверять после правок |

**Как вводить:** текст в чате (правый нижний угол) или 🎤 голос. На шагах с формой — находитесь на указанной странице.

## Smoke-тест API (без UI)

```powershell
# Баланс + график
$r = Invoke-RestMethod -Method Post http://127.0.0.1:8000/api/chat/guest `
  -ContentType "application/json" -Body '{"message":"Сколько на счёте?"}'
$r.charts.Count -ge 1

# Навигация
$r = Invoke-RestMethod -Method Post http://127.0.0.1:8000/api/chat/guest `
  -ContentType "application/json" -Body '{"message":"открой платежи"}'
$r.navigation_path[-1].url   # -> /payments

# Поиск
$r = Invoke-RestMethod -Method Post http://127.0.0.1:8000/api/chat/guest `
  -ContentType "application/json" -Body '{"message":"найди платежи Иванова"}'
$r.message -match "Найдено"

# Заполнение формы
$r = Invoke-RestMethod -Method Post http://127.0.0.1:8000/api/chat/guest `
  -ContentType "application/json" `
  -Body '{"message":"контрагент Федя сумма 5000","page_route":"/payments/instant","form_type":"instant"}'
$r.form_actions | Where-Object { $_.value -eq "Федя" }

# График
$r = Invoke-RestMethod -Method Post http://127.0.0.1:8000/api/chat/guest `
  -ContentType "application/json" -Body '{"message":"Покажи график"}'
$r.charts.Count -ge 1

# Информационный запрос
$r = Invoke-RestMethod -Method Post http://127.0.0.1:8000/api/chat/guest `
  -ContentType "application/json" -Body '{"message":"информационный запрос"}'
$r.navigation_path[-1].url    # -> /other/info-requests
```

## Регрессии 🔴 (проверять после каждого релиза)

- [ ] «информационный запрос» → `/other/info-requests`, не выписка и не default-ответ
- [ ] «Открой все документы с февраля по май» → фильтр периода, не сброс фильтров
- [ ] «с контрагентом ООО ромашка» / «с суммой от 50 руб» → банковский поиск, не page_actions
- [ ] «Открой документ номер 97» → карточка №97
- [ ] Многословная фраза на форме instant — контрагент не захватывает всю строку
- [ ] «со счета крутой» / «добрый счет» — выбор счёта плательщика по названию
- [ ] TTS не зависает после «создай платёжное поручение»
- [ ] После 🎤-отправки поле ввода очищается
- [ ] `format_search_response` не падает при пустых hits
